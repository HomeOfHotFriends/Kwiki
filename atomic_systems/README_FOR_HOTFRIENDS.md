# Fantasy Languages: The HotFriends Edition

Welcome, HotFriends! This README is your personal, living index and guide to all fantasy languages and atomic systems as they develop in this project. Only you get this view.

---

## What is a Fantasy Language?
A fantasy language here is a simulated logic or playful system—no compiler, no VM, just JSON files describing:
- Binary or poetic logics
- Playful toggles (sensible/silly, continuous/discrete, etc.)
- Generative instructions (riff, invert, echo, etc.)
- Anything you can imagine as a "language" for creative computation

---

## Current Fantasy Languages & Atomic Systems

### zero_one
- **Description:** Binary playground. 0 and 1 can mean anything: true/false, continuous/discrete, sensible/silly, etc.
- **Meanings:**
  - 0 = false, 1 = true
  - 0 = continuous, 1 = discrete
  - 0 = sensible, 1 = silly
  - 0 = off, 1 = on
  - 0 = no, 1 = yes

### poemachine
- **Description:** Each instruction is a poetic transformation. No compilation, just interpretation of meaning.
- **Instructions:** invert, riff, pause, echo

### continuous_discrete
- **Description:** 0 means continuous, 1 means discrete. Simulates analog/digital boundaries.
- **Examples:** 0 = gradient, 1 = step

### sensible_silly
- **Description:** 0 means sensible, 1 means silly. Playful toggle for logic modes.
- **Examples:** 0 = rational, 1 = absurd

### true_discrete
- **Description:** 0 means true, 1 means discrete. Paradoxical or playful logic.
- **Examples:** 0 = always, 1 = sometimes
