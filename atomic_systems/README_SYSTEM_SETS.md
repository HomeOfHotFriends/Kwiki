# System Sets & Software Types

This document collates atomic systems into sets, forming the basis for new software types and design docs. Each set is a curated combination of fantasy languages and logic systems, ready to inspire new generative architectures.

---

## System Set: Binary Playground
- **Includes:** zero_one, sensible_silly, true_discrete
- **Purpose:** Explore all possible meanings of binary logic. Useful for toggling modes, playful computation, and creative constraints.

## System Set: Poetic Computation
- **Includes:** poemachine, sensible_silly, continuous_discrete
- **Purpose:** Generative text, poetic logic, and playful toggling. Ideal for creative writing tools, generative poetry, or narrative engines.

## System Set: Analog/Digital Bridge
- **Includes:** continuous_discrete, zero_one, true_discrete
- **Purpose:** Simulate boundaries between analog and digital, continuous and discrete. Useful for signal processing metaphors, art tools, or educational software.

## System Set: Paradox Engine
- **Includes:** true_discrete, zero_one, poemachine
- **Purpose:** Embrace contradiction and paradox. For experimental logic, philosophy engines, or games that break the rules.

# How to Create a New System Set
1. Choose 2–5 atomic systems from atomic_systems/.
2. Define the purpose and possible applications.
3. List included systems and their roles.
4. Optionally, write a design doc (see below).

---

# Example Design Doc: "Narrative Engine"
- **System Set:** Poetic Computation
- **Description:** A tool for generative storytelling using poetic logic and playful toggles.
- **Core Systems:** poemachine (text transformation), sensible_silly (mode toggle), continuous_discrete (narrative pacing)
- **Possible Features:**
  - Riff and echo story beats
  - Switch between rational and absurd plotlines
  - Adjust pacing from smooth to abrupt
